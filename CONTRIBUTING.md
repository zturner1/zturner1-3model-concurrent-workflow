# Contributing

Thanks for your interest in contributing to the 3-Model Concurrent Workflow project!

## Ways to Contribute

### 1. Share Your Workflow Improvements

Have you found better ways to use the 3-model architecture? Open an issue or PR with:
- New output styles (`.styles/`)
- Improved context file templates
- Automation scripts (`scripts/tools/`)
- Prompt templates (`scripts/prompts/`)

### 2. Documentation

- Fix typos or clarify instructions
- Add examples of real-world usage
- Translate documentation
- Improve the architecture layout

### 3. Tool Integration

- Add support for additional AI CLI tools
- Improve install/run scripts for other platforms (Linux, macOS)
- Create shell script equivalents of the batch files

## How to Contribute

### Quick Contributions

1. Fork the repository
2. Make your changes
3. Submit a Pull Request

### Larger Changes

1. Open an issue first to discuss
2. Fork and create a feature branch
3. Make your changes
4. Test with the 3-model workflow
5. Submit a Pull Request

## Commit Message Format

Follow the existing convention:

```
[type] Brief description

- Detail 1
- Detail 2
```

Types:
- `[meta]` - Project structure, config
- `[docs]` - Documentation
- `[style]` - Output style files
- `[script]` - Batch/shell scripts
- `[context]` - Context file templates

## Code of Conduct

- Be respectful and constructive
- Focus on the work, not the person
- Assume good intentions

## Questions?

Open an issue with the `question` label.
